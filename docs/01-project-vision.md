# Project Vision

## Название проекта

Интерактивная мульти-гостиничная система бронирования.

## Цель проекта

Создать универсальный движок для малых гостиниц, гостевых домов, баз отдыха и мини-отелей.

Система должна позволять гостю визуально выбрать гостиницу, этаж, номер, койко-место, просмотреть фото, видео, план комнаты и отправить заявку на бронирование.

## Основная идея

Проект строится не как один сайт под одну гостиницу, а как универсальная система, где каждая гостиница настраивается через данные.

Одна кодовая база должна поддерживать разные гостиницы с разной структурой:

- разное количество этажей;
- разное количество номеров;
- разные схемы этажей;
- разные комнаты;
- разные кровати;
- разную мебель;
- разные тарифы питания;
- разные медиа-файлы.

## Целевая аудитория

- малые гостиницы;
- гостевые дома;
- мини-отели;
- хостелы;
- базы отдыха;
- частные объекты размещения.

## Основной пользовательский сценарий

1. Пользователь открывает сайт.
2. Выбирает гостиницу.
3. Видит здание и общую информацию.
4. Открывает этаж.
5. Видит план этажа: коридор и номера.
6. Кликает по номеру.
7. Смотрит фото, видео и план комнаты.
8. Видит мебель и доступные кровати.
9. Выбирает номер целиком или койко-место.
10. Выбирает тариф питания.
11. Отправляет заявку на бронирование.

## Основной административный сценарий

Администратор должен иметь возможность:

- создавать гостиницы;
- добавлять этажи;
- добавлять номера;
- задавать расположение номеров на плане этажа;
- добавлять мебель в справочник;
- расставлять мебель внутри комнаты;
- добавлять фото, видео и планы;
- управлять статусами номеров;
- управлять статусами койко-мест;
- настраивать тарифы питания;
- просматривать заявки на бронь.

## Важный принцип

Все данные должны быть вынесены из интерфейса.

Интерфейс не должен знать заранее:

- сколько гостиниц;
- сколько этажей;
- сколько номеров;
- сколько кроватей;
- какая мебель есть в номере.

UI должен строиться только на основании данных.

## Хранение данных

На первом этапе данные могут храниться в JSON / JS-файлах.

В будущем система должна быть готова к переходу на базу данных.

Для недорогого обычного хостинга приоритетный вариант будущей базы:

- MySQL / MariaDB.

PostgreSQL не используется как обязательная цель, так как он чаще требует VPS или более дорогого тарифа.

## Технический подход

Frontend:

- React;
- Vite;
- компонентная архитектура;
- интерактивные планы этажей и комнат.

Backend в будущем:

- API;
- PHP или Node.js в зависимости от хостинга;
- MySQL / MariaDB для хранения данных.

## MVP

Первая версия должна включать:

- список гостиниц;
- страницу гостиницы;
- план этажа;
- кликабельные номера;
- карточку номера;
- фото номера;
- видео номера;
- план комнаты;
- список мебели;
- выбор номера или койко-места;
- выбор питания;
- форму заявки.

## Долгосрочная цель

Создать не одиночный сайт, а основу для продукта, который можно адаптировать под разные гостиницы.
## Титульная страница гостиницы

Первая страница гостиницы должна быть выполнена как атмосферный fullscreen-экран.

Фоном используется фото гостиницы, реки, набережной, леса или другого характерного места.

На титульной странице отображаются:

- название гостиницы;
- короткое описание;
- ключевые преимущества;
- стартовая цена;
- кнопка входа в интерактивный выбор.

После нажатия кнопки "Войти" пользователь переходит к выбору этажей.
